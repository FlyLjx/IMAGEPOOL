# Changelog

## Unreleased

## 0.1.18 - 2026-07-14

+ [新增] 首页新增服务器负载卡片，展示 CPU、内存、网络上下行和图片存储磁盘使用情况。
+ [新增] 管理端提供 `/api/system/load` 实时负载接口，首页每 5 秒轻量刷新一次。
+ [调整] 首页移除“账号池状态”和“存储与代理”卡片；运行状况和错误率卡片等高对齐，错误原因列表限制在卡片内滚动。

## 0.1.17 - 2026-07-14

+ [优化] 生图任务采用有界并发调度、账号 FIFO 租约与批量状态持久化，提升高并发下的公平性与吞吐。
+ [修复] ChatGPT Web 生图准备、生成和跨账号重试使用固定 300 秒总预算；已提交会话仍可真实等待 180 秒。
+ [修复] OAuth 凭证失效会删除账号并重试；接口、SSE 与任务日志不再暴露上游 `token_revoked` 响应体。
+ [修复] 图片缓存下载、远程编辑图引用、PoW 与 Turnstile 回退增加超时、取消和并发保护。

## 0.1.16 - 2026-07-13

+ [优化] 生图调度优先使用刚导入账号池的账号；同一批新账号优先分配尚未跑过图的账号，避免重复压同一个账号。

## 0.1.15 - 2026-07-13

+ [调整] 单账号 ChatGPT 生图轮询上限从 60 秒延长至 180 秒，已有的 60 秒旧配置在升级后自动迁移为 180 秒。

## 0.1.14 - 2026-07-13

+ [修复] 最新版本检测同时查询 GitHub Release 与 jsDelivr 并选择较新版本，避免 jsDelivr 版本索引延迟导致前台收不到升级提示。

## 0.1.13 - 2026-07-13

+ [修复] 图片请求遇到 HTTP 401、`token_revoked` 等认证失败时立即删除失效账号，单个任务最多连续切换账号重试 10 次。

## 0.1.12 - 2026-07-13

+ [修复] 图片请求遇到明确的 401 `token_revoked` 时删除失效账号并切换账号重试，单个任务最多触发 1 次认证重试，即使错误发生在普通尝试上限的最后一次。
+ [修复] 取消默认 120 秒任务总截止时间；单个账号会话轮询超时后立即释放并切换账号，后续账号获得完整处理时间。
+ [优化] 无图片引用的 SSE 空流等待缩短为 8 秒且最多恢复 1 次，更早进入会话轮询。
+ [新增] 失败任务保留账号尝试次数、使用账号数和失败账号数，便于确认重试是否执行。

## 0.1.11 - 2026-07-13

+ [修复] 版本检查改用 jsDelivr 的最新 Release tag 解析接口，避免 GitHub 未认证 API 每小时 60 次的共享限流；GitHub API 仅作为失败兜底。
+ [修复] 更新日志按已发布版本读取固定 tag 内容，避免主分支内容与已完成镜像版本不一致。

## 0.1.10 - 2026-07-13

+ [修复] 普通生图不再重复执行账号 bootstrap、Sentinel 和额度预检，账号分配后立即进入实际生图；实际请求返回失效凭证时仍自动删除账号。
+ [修复] SSE 无图片结果时限制空闲等待和恢复次数，提前转入会话轮询；连接 `broken pipe` 和 `unexpected EOF` 会继续轮询，不再直接返回 502。
+ [优化] 任务进度改为按任务批量异步写入 PostgreSQL，自动迁移旧版全量任务文档，避免高并发下反复重写数 MB JSONB 阻塞请求。
+ [调整] 设置页移除已停用的生图账号预检间隔、并发和超时配置。

## 0.1.9 - 2026-07-13

+ [修复] 对齐当前 ChatGPT Web 生图协议：三阶段 prepare 串联 conduit token，复用 parent/trace，支持异步图片任务和 `/f/conversation/resume` 续传。
+ [修复] 按标准 SSE 事件读取图片结果；初始请求及续传遇到网络错误、408/409/425/429/502/504 时在同一账号内退避重试，避免无效换号和 300/600 秒等待。
+ [修复] 单会话轮询限制为 60 秒，图片请求默认总超时 120 秒；取消、内容策略拒绝和真实超时分别返回正确状态，不再统一记为 502。
+ [优化] 定时账号刷新默认调整为 60 分钟并使用轻量校验；导入和手动刷新仍执行完整验证并删除失效账号。
+ [优化] SSE 接口关闭 Nginx 响应缓冲，图片引用解析失败时保留具体上游诊断。
+ [新增] 任务队列表格和实时状态弹窗显示任务创建时间，提交时间与更新时间分开展示。

## 0.1.1 - 2026-07-12

+ [重构] 新增独立 Go 服务 IMAGE POOL，使用全新的数据目录，不迁移原项目数据。
